# マニアプロデュース様 お知らせ被りチェック＆作成スキル

## 使い方

```
店舗名：〇〇マニア 〇〇店
お知らせ素材：
（テキスト貼り付け、または URL、または内容の説明）
```

---

## Step 1: 入力確認

以下の2つが揃っているか確認する。不足があればユーザーに問い合わせる。

| 必要情報 | 例 |
|---|---|
| **店舗名** | 小籠包マニア 中目黒本店 |
| **お知らせ素材** | テキスト・URL・媒体の説明、いずれか |

素材の種類ごとの扱い：

| 種類 | 対応 |
|---|---|
| テキスト | そのまま使用 |
| URL | Step 1.5 でWebFetchして内容を取得 |
| 画像・動画の説明 | ユーザーの説明をそのまま素材として使用 |

## Step 1.5: URL素材の取得（URLが提示された場合のみ）

WebFetchツールでURLの内容を取得する。

- 取得できた場合：本文テキストをお知らせ素材として使用する
- ログイン壁などで取得できなかった場合：その旨を明示してユーザーにテキストを求める
- リトライは1回のみ

### URLの種類による扱い分け

| URLの種類 | 扱い |
|---|---|
| **公式サイト**（各店舗ドメイン） | 内容をそのまま素材として使用してよい |
| **外部メディア**（食べログ・Google・口コミサイト等） | 話題のヒントとして使うが、記事で紹介するメニュー・商品は必ず公式メニューページで実在を確認してから使用する |

**外部URLの場合の追加手順：**
1. 外部URLから「記事にできそうなテーマ（メニュー名・特徴）」を抽出する
2. 店舗の公式メニューページ（`/food` または `/course`）をWebFetchして、そのメニューが実在するか確認する
3. 確認できたメニューのみ記事の主役にする。確認できない場合は別のテーマを選ぶ

## Step 1.6: 素材の品質チェック

記事生成前に、以下の観点で素材の内容を確認する。**問題がある情報は使用しない。**

| チェック項目 | NG例 | 対処 |
|---|---|---|
| **時制の不自然さ** | 現在が2026年なのに「2024年に〜されました！」と鮮度切れの情報をそのまま使う | 使用しない、または「〜の実績を持つ」等の現在形に言い換える |
| **未確認メニューの紹介** | 食べログ等にあるが公式メニューページで確認できないメニュー | 記事の主役にしない |
| **素材にない事実の追加** | キャプションやURLに書かれていない価格・日付・メニュー名を加える | 絶対に追加しない |
| **過度に古い受賞・実績** | 2年以上前の出来事を「〜されました！」と現在進行形で紹介 | 使用しないか「〜の実績を誇る」等に言い換え |
| **脇役メニューの主役化** | お通し・箸休め・汎用サイドを記事の主役にする | 別のメニューに差し替える |

### メニューの主役判定基準

記事の主役にしてよいメニューかどうかは以下で判断する。

**主役にしてよい（以下のいずれかひとつでもあればOK）**
- 「おすすめ」「名物」「イチオシ」「スタッフおすすめ」等の推薦ワードがある
- 具体的な調理法・食感・味・こだわりの説明がある（詳細記述）
- この客先の過去記事でそのメニューが記事化された実績がある

**主役にしてはいけない（NGサイン）**
- お通し・サービス品・無料提供品（例：ザーサイおかわり無料）
- 「〜と一緒に」「〜の箸休めに」としか登場しない脇役的言及
- メニュー名のみで説明・推薦ワードが一切ない
- どこの飲食店にも必ずある汎用サービス品（ライス・お漬物等）

NGと判定した場合は、同じ素材の中で次に主役にふさわしいテーマを探す。素材内に適切なテーマがなければユーザーに別の素材を求める。

### 情報量の十分性チェック

記事生成前に、素材から以下の情報が揃っているか確認する。

| 必要な情報 | 例 |
|---|---|
| メニュー名（主役） | 「自家製チャーシュー」 |
| 特徴・こだわり・食感 | 箸で切れるほど柔らかい、じっくり煮込んだ etc. |
| 来店を促す一言 | 餃子と一緒に、ぜひお試しください etc. |

**情報が足りない場合は記事を無理に作らず、不足している点をユーザーに質問する。**
ユーザーは追加情報の提供に同意しているため、遠慮なく聞いてよい。

質問の例：
- 「〇〇のこだわりや特徴をもう少し教えていただけますか？」
- 「このメニューはどんな食感・味わいですか？」
- 「別のメニューの素材はありますか？」

## Step 2: Excelデータ読み込み

以下の手順でPythonスクリプトをファイルとして書き出してから実行する。
（Bashのパイプ経由だとUNCパスの日本語が文字化けするため、必ずファイル実行すること）

**Step 2-1: スクリプトを一時ファイルに書き出す（Writeツール使用）**

`C:\Users\ozeki\AppData\Local\Temp\check_mania_news.py` に以下の内容で書き出す：

```python
# -*- coding: utf-8 -*-
import openpyxl, os, glob, sys
sys.stdout.reconfigure(encoding='utf-8')

folder = r'\\192.168.2.241\share\disk1\GATEWING MA\顧客情報\00_マニアプロデュース様_お知らせ'
files = glob.glob(os.path.join(folder, '*_お知らせ内容リスト(2年間).xlsx'))
files = [f for f in files if not os.path.basename(f).startswith('~$')]
if not files:
    print("Excelファイルが見つかりません")
else:
    path = max(files, key=os.path.getmtime)
    print(f"読込ファイル: {os.path.basename(path)}\n")
    wb = openpyxl.load_workbook(path, read_only=True, data_only=True)
    for sheet_name in wb.sheetnames:
        ws = wb[sheet_name]
        print(f"=== {sheet_name} ===")
        for i, row in enumerate(ws.iter_rows(values_only=True)):
            if i == 0:
                continue
            date  = row[0] if len(row) > 0 else ''
            title = row[1] if len(row) > 1 else ''
            body  = row[2] if len(row) > 2 else ''
            if title and str(title) not in ('エラー', '該当なし', 'タイトル'):
                body_short = str(body)[:100].replace('\n', ' ') if body else ''
                print(f"  [{date}] {title} / {body_short}")
        print()
    wb.close()
```

## Step 3: 被りチェック

取得したExcelデータと入力素材を意味レベルで比較する。

### 照合の観点

| 観点 | 確認内容 |
|---|---|
| **テーマ** | 同じメニュー・イベント・変更を扱っているか |
| **時期** | 同じ季節・年のキャンペーンか |
| **固有名詞** | メニュー名・コース名が一致するか |
| **目的** | 紹介・告知・変更など告知の種類が同じか |

### 判定基準

| 判定 | 基準 | 次のアクション |
|---|---|---|
| **被りあり** | 同じ内容・テーマ | 報告して終了。別の投稿を促す |
| **類似あり** | 似ているが異なる可能性あり | 理由を示してユーザーに確認を求める |
| **被りなし** | 類似する内容なし | Step 4（お知らせ生成）へ進む |

## Step 4: お知らせ文の生成（被りなし、またはユーザーが続行を指示した場合）

### 前提ルール

あなたは、マニアプロデュース様の各店舗サイトのお知らせを長年担当しているプロのWebライターです。
過去記事のトーン・構成・言い回しを熟知した上で、提示された素材をもとに新しいお知らせ文を仕上げます。

### 記事の種別を判定する

素材の内容から以下のいずれかに分類して、構成を使い分ける。

| 種別 | キーワード例 |
|---|---|
| **メニュー紹介** | 新メニュー・季節限定・おすすめ料理・ドリンク紹介 |
| **コース紹介** | コース・宴会・飲み放題 |
| **営業案内** | 定休日・臨時休業・営業時間・年末年始 |
| **イベント・キャンペーン** | 周年・フェア・特典・割引 |
| **店舗紹介** | 内装・席・空間・アクセス |

---


### 構成パターン（種別別）・HTML形式

`<p>` `<br>` は使わない。メニュー名と `<a>` リンクテキストに店舗カラーの `<span>` を適用する。

#### メニュー紹介・コース紹介

```
[書き出し]

今回は<strong><span style="color:[カラーコード]">「[メニュー名]」</span></strong>をご紹介いたします。
[特徴・こだわりの説明 2〜3文]
[ぜひこの機会に〜してください 1文]

<a href="[お料理メニューURL]" style="color:[カラーコード]">≫ お料理メニューはこちら</a>

皆さまのご来店を心よりお待ちしております！

※画像はイメージです。
```

※コースの場合は `<a href="[コースメニューURL]" style="color:[カラーコード]">≫ コースメニューはこちら</a>`

#### 営業案内（定休日・休業・時間変更）

```
[書き出し]

■[変更内容の見出し]
[変更の内容・日付]

■[追加事項があれば]
[内容]

引き続き、皆様のご予約・ご来店をスタッフ一同心よりお待ちしております。
```

#### イベント・キャンペーン

```
[書き出し]

[イベントの概要を一文で]

【[詳細の見出し]】
[期間・内容・条件など]

【[次の見出し]】
[内容]

[行動喚起 1文]

皆さまのご来店を心よりお待ちしております！
```

### タイトルのパターン

| 種別 | パターン | 例 |
|---|---|---|
| メニュー紹介 | `【おすすめ】「〇〇」のご紹介` | 【おすすめ】「よだれ鶏」のご紹介 |
| 特徴アピール | `〇〇！「〇〇」のご紹介` | パリパリプリプリ！「揚げワンタン」のご紹介 |
| コース紹介 | `〇〇コースのご紹介` | 虎ノ門限定コースのご紹介 |
| 営業案内 | `〇〇のお知らせ` | 定休日変更のお知らせ |
| 時期付き営業 | `【〇〇〜】〇〇のお知らせ` | 【4/19～】定休日変更のお知らせ |

---

### 文体・表現のルール

**守ること**
- 書き出しと締めは上記の固定パターンを必ず使う
- 事実の捏造禁止：素材にない価格・日付・メニュー名は加えない
- 箇条書き禁止（メニュー紹介時）：自然な文章の中に組み込む
- 日付がある場合は必ず曜日を正確に算出して記載する
- 感覚的な表現を積極的に使う（アツアツ、パリパリ、しっとり、じゅわっと等）
- 1文は読みやすい長さに抑え、すべて敬体（です・ます）で統一する
- `※画像はイメージです。` はメニュー・食べ物の紹介記事の末尾に必ず `<p>` で付ける

**HTMLタグの使い方**
- `<p>` `<br>` は**使わない**
- セクション見出し：`■` または `【】`（テキスト記号）
- メニュー名の強調：`<strong><span style="color:[店舗カラー]">「メニュー名」</span></strong>`
- リンク：`<a href="絶対URL" style="color:[店舗カラー]">≫ テキストはこちら</a>`
- 色付きspan・マーカー装飾：メニュー名・リンクのみに限定する

## Step 6: デスクトップのtxtファイルに保存

記事を生成したら、以下の手順でデスクトップのtxtファイルに追記保存する。

**Step 6-1: 保存スクリプトを一時ファイルに書き出す（Writeツール使用）**

`C:\Users\ozeki\AppData\Local\Temp\save_mania_news.py` に以下の内容を書き出す。
`TITLE`・`BODY`・`STORE` の3変数を実際の内容に置き換えること。

```python
# -*- coding: utf-8 -*-
import os
from datetime import datetime

STORE = "餃子マニア 品川本店"   # ← 店舗名を入れる
TITLE = "タイトルをここに入れる"  # ← タイトルを入れる
BODY  = """
<p>本文HTMLをここに入れる</p>
"""  # ← 本文HTMLを入れる

desktop = os.path.join(os.path.expanduser("~"), "Desktop")
filename = datetime.now().strftime("%Y年%m月") + "_マニアお知らせ.txt"
filepath = os.path.join(desktop, filename)

separator = "=" * 60
timestamp = datetime.now().strftime("%Y/%m/%d %H:%M")

entry = f"""
{separator}
【{STORE}】　{timestamp}
{separator}
■ タイトル
{TITLE}

■ 本文（HTML）
{BODY.strip()}

"""

with open(filepath, 'a', encoding='utf-8') as f:
    f.write(entry)

print(f"保存しました: {filepath}")
```


保存先ファイル名：`C:\Users\ozeki\Desktop\YYYY年MM月_マニアお知らせ.txt`

## 重要ルール（全Step共通）

- 照合はキーワード完全一致ではなく意味・内容レベルで判断する
- 同一内容が複数店舗で見つかった場合はすべて列挙する
- 素材にない事実（価格・日付・固有名詞）を記事に追加しない
- 同ジャンルでも年度・時期が明確に異なる場合は「被りなし」と判定してよい
- ユーザーが「進める」と指示した場合は類似ありでもStep 4へ進む

### 使用禁止パターン（品質NG）

- 食べログ・口コミ等の外部URLにあるメニューを、公式メニューページで未確認のまま紹介する
- 現在時点から見て2年以上前の出来事を「〜されました！」と現在進行形のトーンで紹介する
- 公式サイトに掲載のないコース・価格・特典を記載する
- 素材の文章をそのままコピーして使う（必ずお知らせ文体に書き直す）
